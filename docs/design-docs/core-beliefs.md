# Core Beliefs

- Versioned repo context is the source of truth.
- Security, reliability, and product trust are architectural requirements.
- Prefer simple, inspectable data flows over opaque automation.
- Keep long-running work out of request handlers.
- Preserve tenant isolation at every storage and retrieval boundary.
- Build UI surfaces for repeated operational use, not one-time demos.
- Use Tailwind CSS for local, consistent, accessible component primitives.
